# Frontend Designer

## Overview

This is a professional frontend design skill that helps create beautiful, consistent, and user-friendly interfaces.

## When to Use

- When user asks for UI/UX improvements or design refactoring
- When visual consistency across pages is needed
- When buttons, spacing, colors, or typography need optimization
- When creating or updating CSS stylesheets

## Core Principles

1. **Consistency First**: All pages should follow the same design system
2. **Visual Hierarchy**: Clear information architecture with proper spacing
3. **Responsive Design**: Works on all screen sizes
4. **Accessibility**: Good contrast, clear labels, and semantic HTML

## Design System

### Colors
- Primary: Consistent accent color across all pages
- Neutrals: Harmonious grayscale palette
- Backgrounds: Clean, readable backgrounds

### Typography
- Consistent font families and sizes
- Clear heading hierarchy
- Proper line heights and spacing

### Spacing
- Consistent margin and padding patterns
- Logical section separations
- Proper breathing room between elements

### Buttons
- Consistent height and padding
- Clear hover and active states
- Proper visual weight for primary actions

## Usage Instructions

1. **Analyze Existing Design**: Review current HTML and CSS structure
2. **Identify Issues**: Look for inconsistencies, spacing problems, or visual clutter
3. **Apply Design System**: Update styles to follow consistent patterns
4. **Test Across Pages**: Ensure all pages look and behave consistently
5. **Validate**: Check for accessibility and responsiveness

## Example Workflow

```
1. Read existing HTML and CSS files
2. Identify design inconsistencies
3. Update CSS with consistent design tokens
4. Apply uniform spacing and typography
5. Test all pages to ensure consistency
6. Verify with user feedback
```
