# Supabase 数据库说明

本文档基于当前项目实际连接的 Supabase REST Schema 整理，连接来源为后端环境变量中的 `SUPABASE_URL`。

## 当前表清单

| 表名                         | 当前行数 | 项目是否使用 | 作用                        |
| -------------------------- | ---: | ------ | ------------------------- |
| `api_settings`             |   15 | 是      | 保存后台配置项，例如模型配置、积分规则、应用模式等 |
| `user_points_balances`     |    4 | 是      | 用户积分余额主表                  |
| `user_points_transactions` |    0 | 是      | 用户积分流水表，用于记录积分增减明细        |

| `zpay_transactions`        |   26 | 是      | ZPay 支付订单和订阅订单记录          |

以下 3 张旧表当前项目代码未引用，且清理前行数均为 0：

| 表名                   | 原用途      | 替代方案                   | 清理状态                                             |
| -------------------- | -------- | ---------------------- | ------------------------------------------------ |

| `users`              | 旧版自建用户表  | Supabase Auth          | 已通过 REST API 确认数据为空；表结构需在 Supabase SQL Editor 删除 |

当前本地后端连接的是项目实际使用的 Supabase REST API：`https://spb-kemqk3h0a423q1q5.supabase.opentrust.net`。REST API 可以清空数据，但不能执行 `drop table` 这类 DDL。若要从 Supabase 后台完全删除表结构，请在该项目的 SQL Editor 执行：

```sql
drop table if exists public.verification_codes cascade;
drop table if exists public.user_points cascade;
drop table if exists public.users cascade;
```

## 表结构说明和建表 SQL

### api\_settings

用途：保存应用后台配置。后端通过 `scope + setting_key` 读取配置值，例如 API Key、模型名、积分赠送规则、应用模式等。

| 列名              | 类型          | 含义                       |
| --------------- | ----------- | ------------------------ |
| `id`            | bigint      | 主键，自增 ID                 |
| `scope`         | text        | 配置作用域，当前主要使用 `global`    |
| `setting_key`   | text        | 配置键名，例如 `OPENAI_API_KEY` |
| `setting_value` | text        | 配置值                      |
| `created_at`    | timestamptz | 创建时间                     |
| `updated_at`    | timestamptz | 更新时间                     |

```sql
create table if not exists public.api_settings (
  id bigint generated by default as identity primary key,
  scope text not null default 'global',
  setting_key text not null,
  setting_value text not null default '',
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now(),
  constraint api_settings_scope_key_unique unique (scope, setting_key)
);

create index if not exists api_settings_scope_idx on public.api_settings (scope);

create or replace function public.set_api_settings_updated_at()
returns trigger
language plpgsql
as $$
begin
  new.updated_at = now();
  return new;
end;
$$;

drop trigger if exists api_settings_updated_at on public.api_settings;
create trigger api_settings_updated_at
before update on public.api_settings
for each row
execute function public.set_api_settings_updated_at();

alter table public.api_settings enable row level security;

drop policy if exists "api_settings service role full access" on public.api_settings;
create policy "api_settings service role full access"
on public.api_settings
for all
to service_role
using (true)
with check (true);
```

### user\_points\_balances

用途：用户积分余额主表。后端积分余额、每日领取、新用户赠送、生成扣费都会读写此表。

| 列名                        | 类型          | 含义                                   |
| ------------------------- | ----------- | ------------------------------------ |
| `user_id`                 | text        | 用户 ID，当前保存 Supabase Auth 用户 UUID 字符串 |
| `balance`                 | bigint      | 当前可用积分余额                             |
| `total_earned`            | bigint      | 历史累计获得积分                             |
| `total_spent`             | bigint      | 历史累计消耗积分                             |
| `signup_bonus_awarded_at` | timestamptz | 新用户注册送积分发放时间，非空表示已发放                 |
| `last_daily_claim_at`     | timestamptz | 最近一次领取每日免费积分时间                       |
| `created_at`              | timestamptz | 创建时间                                 |
| `updated_at`              | timestamptz | 更新时间                                 |

```sql
create table if not exists public.user_points_balances (

  balance bigint not null default 0,

  signup_bonus_awarded_at timestamptz,
  last_daily_claim_at timestamptz,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now(),
  constraint user_points_balances_balance_nonnegative check (balance >= 0),
  constraint user_points_balances_total_earned_nonnegative check (total_earned >= 0),
  constraint user_points_balances_total_spent_nonnegative check (total_spent >= 0)

before update on public.user_points_balances

execute function public.set_user_points_balances_updated_at();
```

### user\_points\_transactions

用途：积分流水表。用于记录用户每次积分增加、扣减、退款等明细。当前线上表为空，但代码中直接扣分兜底逻辑会尝试写入流水。

| 列名                 | 类型          | 含义                                                     |
| ------------------ | ----------- | ------------------------------------------------------ |
| `id`               | uuid        | 主键，默认 `gen_random_uuid()`                              |
| `user_id`          | uuid        | 用户 ID                                                  |
| `change_amount`    | integer     | 本次积分变化量，正数为增加，负数为扣减                                    |
| `balance_after`    | integer     | 变化后的积分余额                                               |
| `transaction_type` | text        | 流水类型，例如 `consume`、`daily_free`、`signup_bonus`、`refund` |
| `reason`           | text        | 流水原因说明                                                 |
| `metadata`         | jsonb       | 附加信息，例如业务来源、生成数量等                                      |
| `created_at`       | timestamptz | 创建时间                                                   |

```sql
create table if not exists public.user_points_transactions (
  id uuid primary key default gen_random_uuid(),
  user_id uuid not null,
  change_amount integer not null,
  balance_after integer not null,
  transaction_type text not null,
  reason text not null default '',
  metadata jsonb not null default '{}'::jsonb,
  created_at timestamptz not null default timezone('utc'::text, now()),
  constraint user_points_transactions_change_amount_nonzero check (change_amount <> 0)
);

create index if not exists user_points_transactions_user_id_created_at_idx
on public.user_points_transactions (user_id, created_at desc);

create index if not exists user_points_transactions_type_idx
on public.user_points_transactions (transaction_type);

alter table public.user_points_transactions enable row level security;

drop policy if exists "user_points_transactions service role full access" on public.user_points_transactions;
create policy "user_points_transactions service role full access"
on public.user_points_transactions
for all
to service_role
using (true)
with check (true);
```

### generation_tasks

用途：保存用户图片生成任务状态。套图和服饰生成会先创建任务，前端保存 `task_id` 并通过状态接口轮询；页面刷新后可继续恢复生成结果。

| 列名 | 类型 | 含义 |
| --- | --- | --- |
| `id` | text | 生成任务 ID，主键 |
| `user_id` | uuid | 任务所属用户 |
| `mode` | text | 生成模式，例如 `suite`、`fashion` |
| `request_id` | text | 积分扣减请求 ID |
| `status` | text | 任务状态：`pending`、`running`、`succeeded`、`failed` |
| `result` | jsonb | 生成成功后的结果 JSON |
| `error` | text | 失败错误摘要 |
| `details` | text | 失败详情 |
| `spend_record` | jsonb | 本次扣分记录，用于失败自动退款 |
| `refunded` | boolean | 是否已自动退款 |
| `refund_error` | text | 自动退款失败原因 |
| `created_at` | timestamptz | 创建时间 |
| `updated_at` | timestamptz | 更新时间 |

迁移文件：

```text
supabase/migrations/20260429_create_generation_tasks.sql
```

RLS 策略：当前仅开放 `service_role` 全权限访问。前端不能直接读写该表，必须通过后端接口按登录用户校验任务归属。

### zpay\_transactions

用途：ZPay 支付订单表。后端创建支付订单、支付回调、订阅续期、会员状态查询都会使用此表。

| 列名                 | 类型          | 含义                                  |
| ------------------ | ----------- | ----------------------------------- |
| `id`               | uuid        | 主键，默认 `gen_random_uuid()`           |
| `out_trade_no`     | varchar     | 商户订单号，业务侧唯一订单号                      |
| `user_id`          | uuid        | 用户 ID                               |
| `amount`           | numeric     | 支付金额                                |
| `status`           | text        | 订单状态，当前代码使用 `pending`、`success` 等   |
| `type`             | text        | 支付类型，例如 `one_time` 或 `subscription` |
| `product_id`       | text        | 商品或套餐 ID                            |
| `subscribe_start`  | timestamptz | 订阅开始时间                              |
| `subscribe_expire` | timestamptz | 订阅到期时间                              |
| `trade_no`         | text        | ZPay 平台交易号                          |
| `created_at`       | timestamptz | 创建时间                                |
| `updated_at`       | timestamptz | 更新时间                                |

```sql
create table if not exists public.zpay_transactions (
  id uuid primary key default gen_random_uuid(),
  out_trade_no varchar not null unique,
  user_id uuid,
  amount numeric not null,
  status text not null default 'pending',
  type text not null,
  product_id text,
  subscribe_start timestamptz,
  subscribe_expire timestamptz,
  trade_no text,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now(),
  constraint zpay_transactions_amount_positive_check check (amount > 0),
  constraint zpay_transactions_type_check check (type in ('one_time', 'subscription'))
);

create index if not exists zpay_transactions_user_id_idx on public.zpay_transactions (user_id);
create index if not exists zpay_transactions_status_idx on public.zpay_transactions (status);
create index if not exists zpay_transactions_product_id_idx on public.zpay_transactions (product_id);
create index if not exists zpay_transactions_subscribe_expire_idx on public.zpay_transactions (subscribe_expire);

create or replace function public.set_zpay_transactions_updated_at()
returns trigger
language plpgsql
as $$
begin
  new.updated_at = now();
  return new;
end;
$$;

drop trigger if exists zpay_transactions_updated_at on public.zpay_transactions;
create trigger zpay_transactions_updated_at
before update on public.zpay_transactions
for each row
execute function public.set_zpay_transactions_updated_at();

alter table public.zpay_transactions enable row level security;

drop policy if exists "Authenticated users can read own zpay transactions" on public.zpay_transactions;
```
